# Design: Robust Shopping Cart Retry and Error Filtering

This document details the architecture and design patterns for Stage 2 (Design) of the cart mutation robustness task.

## 1. Rust-side Parameter Translation Shim

To handle flat argument representations generated by the LLM when executing cart mutations, we introduce a parameter translation layer in `mcp_client.rs`.

### Target Methods
- `mcp_client::McpClientManager::call_tool_resolved`

### Detection Trigger
The translation is triggered if:
1. `requested_name` contains `"update_cart"`, `"add_to_cart"`, or `"remove_from_cart"`.
2. The arguments `args` is a JSON Object (`serde_json::Value::Object`).
3. The arguments Object does NOT contain the key `"items"`.

### Translation Rules
When triggered, the shim identifies and extracts the following keys:
- **Item-level Identifiers**: `"productId"`, `"product_id"`, `"variantId"`, `"variant_id"`, `"skuId"`, `"sku_id"`, `"id"`.
- **Item-level Quantities**: `"quantity"`, `"qty"`.

If any of these fields are found, they are removed from the top-level arguments Object and inserted into a nested item Object:
```json
{
  "productId": "<value>",
  "quantity": <value>
}
```
This nested item Object is wrapped in a JSON Array and inserted under the key `"items"` of the top-level arguments Object. Other metadata fields (e.g. `deviceId`, `addressId`, etc.) are left intact at the top-level.

## 2. Prompt Hardening Directives

We will update the system prompt (`system_prompt.txt`) and `shopping_assistant` role configurations (`agent_roles.json`) to enforce human-friendly communications.

### Core Directives
1. **Remove legacy formatting rules**: Completely strip the instruction `For errors: Failed: reason. Next: ...`.
2. **Add error-recovery instructions**: Explicitly instruct the LLM to never leak raw JSON payloads or stack traces. If a tool fails, the agent must check the tool's expected schema, correct any format issues, and retry up to 3 times internally.
3. **Natural human responses**: Require all responses (including errors or retries) to be expressed in friendly, non-technical human language.
